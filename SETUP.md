## Part 1 — Local Development (write code here)

### Prerequisites
- Node.js >= 20
- pnpm >= 9 → `npm install -g pnpm@9`
- Python >= 3.11
- Docker Desktop (for local Postgres + Redis)

```bash
# 1. Clone
git clone git@github.com:HG-E/Researchvy.git
cd Researchvy

# 2. Start local database + redis
docker-compose up -d

# 3. Install all dependencies (one command covers all apps + packages)
pnpm install

# 4. Configure environment
cp .env.example .env
# Fill in .env — minimum required:
#   JWT_SECRET     → openssl rand -hex 64
#   ENCRYPTION_KEY → openssl rand -hex 32
#   ORCID_CLIENT_ID / ORCID_CLIENT_SECRET
#     → register free at: https://sandbox.orcid.org/developer-tools

# 5. Set up database
pnpm db:generate     # Generate Prisma client
pnpm db:migrate      # Create tables
pnpm db:seed         # Create demo account

# 6. Run everything
pnpm dev             # API on :3001 + Web on :3000
```

**Separate terminal — Python worker:**
```bash
cd workers
pip install -r requirements.txt
python worker.py
```

Demo login: `demo@researchvy.com` / `demo1234`
API docs: http://localhost:3001/docs

## Part 2 — Production Deployment (go live)

### Step 1 — Database: Neon (free Postgres, forever)

1. Go to [neon.tech](https://neon.tech) → Sign up free
2. New Project → Name: `researchvy` → Region: closest to your users
3. Copy the **Connection String** — looks like:
   ```
   postgresql://user:password@ep-cool-name-123.us-east-2.aws.neon.tech/researchvy?sslmode=require
   ```
4. Save it — you'll paste this as `DATABASE_URL` in Render

---

### Step 2 — API + Worker: Render (free hosting)

1. Go to [render.com](https://render.com) → Sign up with GitHub
2. New → **Blueprint** → Connect `HG-E/Researchvy`
3. Render reads `render.yaml` and creates both services automatically
4. Fill in the environment variables in the Render dashboard:

**For `researchvy-api`:**
```
DATABASE_URL      → paste your Neon connection string
ENCRYPTION_KEY    → openssl rand -hex 32  (run this locally)
ORCID_CLIENT_ID   → from orcid.org/developer-tools
ORCID_CLIENT_SECRET → from orcid.org/developer-tools
ORCID_REDIRECT_URI → https://api.yourdomain.com/api/v1/auth/orcid/callback
OPENALEX_EMAIL    → your real email
NEXTAUTH_URL      → https://yourdomain.com
```

**For `researchvy-worker`:**
```
DATABASE_URL      → same Neon connection string
OPENALEX_EMAIL    → your real email
```

> `JWT_SECRET` is auto-generated by Render — no action needed.

---

### Step 3 — Frontend: Vercel (free, global CDN)

1. Go to [vercel.com](https://vercel.com) → Sign up with GitHub
2. New Project → Import `HG-E/Researchvy`
3. **Root Directory** → set to `apps/web`
4. Framework: Next.js (auto-detected)
5. Add environment variable:
   ```
   NEXT_PUBLIC_API_URL → https://researchvy-api.onrender.com
   ```
   (or `https://api.yourdomain.com` after DNS is set up)
6. Deploy

---

### Step 4 — Keep-alive: UptimeRobot (free, prevents cold starts)

Render's free tier sleeps after 15 min idle → 30s cold start.
UptimeRobot pings every 14 min → API stays awake.

1. Go to [uptimerobot.com](https://uptimerobot.com) → Free account
2. Add Monitor → HTTP(s)
3. URL: `https://researchvy-api.onrender.com/ping`
4. Interval: **14 minutes**
5. Done — your API now responds instantly, always

---

### Step 5 — Redis: Upstash (free, optional for v1)

Redis is wired but not critical in v1. Add when ready:

1. [upstash.com](https://upstash.com) → Create Database → Region: same as Render
2. Copy the `REDIS_URL` → add to `researchvy-api` env vars in Render

## Part 3 — Connect Your Domain

Your domain works with **any registrar** — Namecheap, GoDaddy, Cloudflare,
Google Domains, Porkbun, or any other. The DNS records are identical regardless.

### Where to find DNS settings

| Registrar | Path |
|---|---|
| Namecheap | Dashboard → Domain List → Manage → Advanced DNS |
| GoDaddy | My Products → DNS → Manage |
| Cloudflare | Select domain → DNS → Records |
| Google Domains | My Domains → Manage → DNS |
| Porkbun | Domain Management → DNS |
| Any other | Look for "DNS Management", "DNS Records", or "Name Servers" |

---

### DNS Records to Add

After deploying, you'll have these URLs from Render and Vercel:
- Vercel gives you: `researchvy.vercel.app`
- Render gives you: `researchvy-api.onrender.com`

Add these records in your domain registrar's DNS panel:

```
Type     Name    Value                           TTL      Purpose
──────   ────    ─────────────────────────────   ──────   ───────────────────────────
A        @       76.76.21.21                     Auto     yourdomain.com → Vercel
CNAME    www     cname.vercel-dns.com            Auto     www.yourdomain.com → Vercel
CNAME    api     researchvy-api.onrender.com     Auto     api.yourdomain.com → Render
```

**Then in Vercel:**
Settings → Domains → Add `yourdomain.com` and `www.yourdomain.com`
Vercel auto-provisions SSL (HTTPS) — no extra steps.

**Then in Render:**
Your service → Settings → Custom Domains → Add `api.yourdomain.com`
Render auto-provisions SSL too.

**DNS propagation:** Changes take 1–48 hours globally (usually under 30 min).
Check progress at: [dnschecker.org](https://dnschecker.org)

---

### After DNS is live — update these env vars

In Render (`researchvy-api`):
```
ORCID_REDIRECT_URI → https://api.yourdomain.com/api/v1/auth/orcid/callback
NEXTAUTH_URL       → https://yourdomain.com
```

In Vercel (`researchvy-web`):
```
NEXT_PUBLIC_API_URL → https://api.yourdomain.com
```

## Common Commands

```bash
pnpm dev              # Start API (:3001) + Web (:3000)
pnpm build            # Full production build
pnpm typecheck        # TypeScript check across all packages
pnpm lint             # ESLint across all packages
pnpm test             # Run API tests

pnpm db:migrate       # Apply pending DB migrations
pnpm db:studio        # Prisma Studio — visual DB editor at :5555
pnpm db:seed          # Populate demo data
pnpm db:reset         # Wipe + re-migrate (local dev only — never run in production)
```
